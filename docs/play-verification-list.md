# 実プレイ確認リスト

総合ルール ver.1.06 を前提とした、シミュレータの手動確認用チェックリストです。  
自動検証（`verify-ability-*` / `audit-behavioral-risks` / **`verify-play-checklist.mjs` 37カード**）は **2026-06-28 通過済み**です。実盤面での最終確認は任意です。

- 正本: [総合ルール PDF](https://llofficial-cardgame.com/wordpress/wp-content/uploads/2026/04/28140005/LoveLiveTCG_cr_1.06_260428.pdf)
- 複雑カード全件（108件）: [complex-risky-cards.md](./complex-risky-cards.md)
- デュアル盤監査 JSON: [dual-mode-gap-audit.json](./dual-mode-gap-audit.json)
- **相手盤面効果レジストリ（全件）**: [opponent-board-effects-registry.md](./opponent-board-effects-registry.md)（`node scripts/audit-opponent-board-effects.mjs` で再生成）
- **対戦モード実装計画（bp2/bp3 テスト含む）**: [versus-mode-implementation-plan.md](./versus-mode-implementation-plan.md)

## 使い方

1. **A → B → C** の順で確認するのがおすすめです。
2. 各項目の `[ ]` を `[x]` に変えて進捗を記録してください。
3. 不具合があれば「結果」欄にメモし、カード番号をチャットで報告してください。
4. **ソロ対戦**（相手盤入力）と**デュアル盤**（2画面）で挙動が変わるカードは両方試すとよいです。

## 凡例

| 優先度 | 意味 |
|--------|------|
| **A** | ユーザー報告 or 直近コード修正あり。**回帰確認最優先** |
| **B** | 相手盤参照あり。デュアル盤で確認済み。**オンライン対戦も Phase 3–5 で対応**（公開盤 read + mutate/choice プロトコル + 常時追従）。一部の複雑カードのみ手動指示（[versus-online-known-gaps.md](./versus-online-known-gaps.md)） |
| **C** | 効果が複雑（リスクスコア ≥ 6）。分類は済んでいるが段階解決の確認が必要 |

## 共通チェック観点（総合ルール）

| ルール | 確認すること |
|--------|-------------|
| **5.9.1** | 「Eを支払う」= エネルギー置き場のカードを**ウェイト**にする（枚数は増えない） |
| **5.9.1** | 「エネルギーをアクティブにする」= 既存ウェイトを**復帰**（デッキから新規配置しない） |
| **8.4 / 11.6** | ライブ成功時 = ライブ枠上で解決**後**に成功ライブ置き場へ移動（8.4.4 → 8.4.7） |
| **11.10 / 11.11** | ポジションチェンジ（1人）とフォーメーションチェンジ（複数同時配置）は別 |
| **9.1.3** | 実行不能な指示は行わない／可能な限り実行 |
| **1.3.1** | カード文と総合ルールが矛盾する場合は**カード文優先** |
